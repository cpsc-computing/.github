# Security Policy

## Supported Versions

| Repository | Version | Supported |
|------------|---------|-----------|
| cpsc-core | 0.x | ✅ |
| cpsc-engine-python | 0.x | ✅ |
| cpsc-engine-rtl | 0.x | ✅ |
| cpac-engine-python | 1.x | ✅ |

## Reporting a Vulnerability

If you discover a security vulnerability in any CPSC repository, please report it responsibly:

1. **Do NOT** open a public GitHub Issue
2. **Email** security concerns to: `security@bitconcepts.tech` (or your preferred contact)
3. **Include**:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 7 days
- **Resolution Target**: Depends on severity

## Scope

Security concerns relevant to CPSC include:

- **cpac-engine-python**: Encryption layer vulnerabilities, key handling issues
- **cpsc-engine-rtl**: Hardware security considerations
- **All repos**: Supply chain, dependency vulnerabilities

## Disclosure Policy

We follow coordinated disclosure:

- We will work with reporters to understand and address issues
- We will credit reporters (unless anonymity is requested)
- Public disclosure after a fix is available
