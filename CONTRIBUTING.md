## How to Contribute

### Reporting Issues

- Use GitHub Issues in the relevant repository
- Include clear reproduction steps
- Specify which repository and version you're using

### Suggesting Enhancements

- Open a GitHub Issue with the `enhancement` label
- Describe the use case and proposed solution
- Reference relevant specification sections if applicable

### Code Contributions

1. **Fork** the repository
2. **Create a branch** for your changes
3. **Follow existing code style** and conventions
4. **Write tests** where applicable
5. **Submit a Pull Request** with a clear description

### Documentation Contributions

Documentation improvements are welcome. Please ensure:

- Consistency with existing terminology (see `GLOSSARY.md` in cpsc-core)
- References to specifications use correct section numbers
- Examples are tested and working

## Repository-Specific Guidelines

Each repository may have additional guidelines in its own `CONTRIBUTING.md`:

- **cpsc-core** — Specification changes require careful review
- **cpsc-engine-python** — Follow Python best practices, include tests
- **cpsc-engine-rtl** — VHDL style guidelines in `docs/`
- **cpac-engine-python** — Maintain lossless guarantees, benchmark changes

## Questions?

For questions about contributing, open a Discussion or contact BitConcepts, LLC.
